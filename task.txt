REST AP - JSON

/users - Return HTML Data  (logic for our REST API)
/api/users/ - Return JSON Data

GET /users - HTML Document Render

GET /api/users - List all users / Return all users  (also support mobile users) 

Dynamic Path Parameters:
GET /api/users/1 - Get the user with ID 1
GET /api/users/2 - Get th euser with ID 2
               :id --> 1, 2, 3, 4, etc (Dynamic Value/Route/Path)

* Dynamic Path Parameters:
    GET /api/users/:id
* :id --> means its a Dynamic path parameter
* :  ---> Colon means 'Dynamic', ':' colon creates the dynamic route
* id ---> id means a 'Variable', it can be any variable

POST /api/users - Create new user

PATCH /api/users/id - Edit the existing user with ID --> id = 1, 2, 3,...

DELETE /api/users/id - Delete the user with ID --> id = 1, 2, 3, 4...
